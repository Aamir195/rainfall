About The Project

In order to create a innovative and balanced project we have used TRIZ principle, also known as the theory of inventive problem solving. Here are the steps our team has followed:

1.TRIZ:ORGANIZE
Segmentation:
For making our project we have segmented our work into several segment, like Data Collecting, Data Preprocessing, Model Making (using Machine learning) and deployment. Every step is explained beloved.
  • Data Collecting: For making our Project ‘Rainfall Prediction’ we have to collect some data. So, for this we collected our data from government site.
  • Data Preprocessing: After getting the data, we have to do some preprocessing on data. It will be helpful for modeling the better predictive system. Like we have to do work on null/missing value, change some data, data manipulation, etc.
  • Model Making: For making the precise prediction of rainfall we are going to build some accurate model using Machine Learning and using some tool.
  • Deployment: For better visualization and for making easy to use for any user we are going to make a frontend. For creating this we will use Python, HTML, API, CSS, JavaScript etc.
    •
    2.TRIZ:COMPOSE
    Universality:
    To add universality to our project, we will be implementing the following things:
  • We will make our project in such a way that it will give predictions on the basis of the input of the user.
  • We will expand it to make predictions for each state in India.
  • We will also categorize danger or heavy rainfall regions for given inputs and make visualization for the same.
    • In this way our project will able to perform multiple task and suffice different users needs.
    3.TRIZ:PROCESS
  • Continuity of useful Action: As we are going to train a different machine learning model for our project “Rain Fall Prediction”. For all these models we will definitely get different accuracy and precision. After comprising all these models, a model which give higher accuracy and precision, we will continue with that model. From same model we will predict the rainfall prediction, and from the same model we will make a model for a front-end. And we will also change some features for better accuracy. In this way we will continue with the most accurate work.
  • Prior Action: Before making the models, we will do some preprocessing on our data. Which will help in understanding the features and helpful in making decision. We will do some visualization to better understand the data and also useful for making training models which will give better prediction of rainfall.
    •
    4.TRIZ:INTERACTION
  • DYNAMICITY: As known, Dynamicity makes an object or its environment automatically adjust for optimal performance at each stage of operation. The models will make better predictions and also optimize space and time as and when required as we will get new and refurbished data by studying different models. This will make the flow better while reducing errors making the model self-analysed, providing better results.
  • EQUIPOTENTIALITY: The model will follow principles of Equipotentiality from the start. The required data will be processed in the same way without differentiating the order of values hence not decrementing the potential that every data has to make a change in the result or provide with a better one.
    •
    5.TRIZ:SERVICES
  • Mediator: Our project is divided into mainly three parts, frontend, backend and our machine learning model. So here we will be using our backend as a mediator that connects frontend with our machine learning model.
  • Self-service: So, after we have integrated the frontend, backend and the machine learning model, it will be able to work on its on without external intervention. The user will be directly able to get the results by interacting with our service.
